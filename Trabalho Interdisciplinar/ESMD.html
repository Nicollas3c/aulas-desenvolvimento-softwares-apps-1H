estrutura cliente-servidor


arquitetura cliente-servidor

A arquitetura de rede na qual existe um servidor que é o responsável pelo processamento e o armazenamento das informações, e que por sua vez é acessado por computadores que solicitam serviços do servidor é conhecida como cliente-servidor. Normalmente o servidor é responsável por vários processos distintos, criados a partir da solicitação dos clientes. Um cliente solicita serviços do servidor quando estes são processos específicos e só podem ser executados ou armazenados no servidor. Dependendo do tipo de aplicação, um cliente pode interagir com um ou mais servidores. Em geral os servidores tem maior capacidade de processamento do que os clientes, de modo que possam atender as solicitações dos vários clientes simultaneamente. A prioridade de atendimento das requisições dos clientes é controlada pelo aplicativo ou banco de dados, em conjunto com o sistema operacional. Nos servidores, é comum existirem milhares de processos sendo executados simultaneamente e concorrendo pela utilização dos recursos de hardware do servidor como processador, memória e eventualmente por recursos de disco (storage). A maioria dos bancos de dados utiliza esta arquitetura, pois o SGBD é instalado no servidor e os usuários utilizam um software cliente instalado no dispositivo local (notebook, celular, etc.), que seja compatível com o SGBD. Atualmente, o acesso via Cloud Computing, onde o SGBD está instalado na infraestrutura de cloud e o acesso é realizado por meio da Internet também é muito utilizado, porém o conceito de cliente server, bem como suas vantagens são os mesmos.


fluxo de silicitação em uma rede cliente-servidor

  • O cliente possui uma interface de software(client) para o acesso ao servidor
  • O cliente monta a solicitação preenchendo os campos da interface de software e monta as requisições
  • As requisições são enviadas ao servidor
  • O servidor processa as requisições
  • O servidor devolve as requisições processadas para o cliente

Vantagens na utilização da arquitetura Cliente-Servidor

  • Como o cliente tem papel ativo na montagem das requisições, executando inclusive algumas validações básicas por meio da interface de software, o tráfego de rede é reduzido, porque o processamento mais importante é executado no servidor.
  • É comum que os computadores envolvidos na arquitetura cliente-servidor sejam de fabricantes diferentes, e até mesmo utilizem sistemas operacionais diferentes. Por exemplo, é possível que uma estação configurada com o sistema operacional Linux acesse um servidor com o sistema operacional Windows e com um SGBD Oracle, por meio da interface de software (client). O que garante a compatibilidade é a comunicação por meio do protocolo TCP-IP.
  • Redução de custos e a possibilidade de upgrade do servidor, caso haja aumento na utilização dos recursos (escalabilidade).
